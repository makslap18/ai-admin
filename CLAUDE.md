## О проекте

AI-Администратор ("Записали") — мультитенантная SaaS-платформа AI-ботов для автосервисов и детейлинг-студий. Построена на n8n (self-hosted), PostgreSQL + pgvector, LLM (Gemini Flash через OpenRouter). Три канала: Telegram, веб-виджет (iframe), Авито.

## Архитектура — ключевые правила

1. **Мультитенантность**: каждый сервис = отдельная schema `service_N` с views на public-таблицы. Фильтрация всегда по `service_id`. НИКОГДА не делать запросы без фильтра по `service_id`.
2. **Анонимизация ПДн (152-ФЗ)**: реальные данные клиента (имя, телефон, email) НИКОГДА не передаются в LLM. Используются алиасы (`alias_name`, `alias_telegram`). Это юридическое требование.
3. **SQL-безопасность**: ТОЛЬКО параметризованные запросы (`$1`, `$2`). НИКАКОЙ string interpolation в SQL. Это было исправлено в аудите — не регрессировать.
4. **Каналы**: Telegram, Web, Avito. Роутинг по полю `channel` в 02 brain. При добавлении нового канала — добавить ветку в Switch Channel.
5. **Webhook баг n8n**: после обновления WF через API/MCP — деактивация → активация → `docker compose restart n8n`.

## n8n workflows — правила редактирования

- При обновлении ноды через n8n API (`updateNode`) — передавать ВСЕ параметры ноды, не только изменённые. API заменяет parameters целиком.
- Не менять webhookId существующих webhook-нод — это сломает все интеграции.
- errorWorkflow = `3muBfH7DpVipSOpN` (06_error) — подключать ко всем новым WF.
- retryOnFail: true, maxTries: 3 — на всех HTTP-нодах.

## БД — правила

- Новые таблицы: добавлять в `public` schema с `service_id`, затем создавать views в schema каждого сервиса.
- Индексы: на часто запрашиваемые поля (service_id, session_id, status, date).
- EXCLUDE constraint на appointments — не трогать, он предотвращает пересечение слотов.
- `ON CONFLICT` — использовать partial indexes где уместно (см. avito: `WHERE real_avito_id IS NOT NULL`).

## Виджет — правила

- XSS-защита обязательна: `esc()` для HTML, `textContent` вместо `innerHTML`.
- CSP заголовки: `frame-ancestors` динамический из `allowed_origins`.
- Rate limit: 10 msg/min по session_id (уровень WF) + 20 req/min по IP (Traefik).
- Сессия: `localStorage` с ключами `asw_{slug}_*` (не sessionStorage — теряется при обновлении).

## Безопасность — абсолютные запреты

- НИКОГДА не хардкодить токены ботов, API-ключи, пароли в коде или документации (кроме внутренних саммари с пометкой INTERNAL).
- НИКОГДА не отключать SSL-верификацию в production.
- НИКОГДА не использовать `eval()` или `innerHTML` с пользовательским вводом.
- НИКОГДА не логировать персональные данные клиентов (имя, телефон, email) — только алиасы.

## Стиль работы

- Перед изменением кода — прочитать текущую версию файла.
- Минимальные изменения: не рефакторить код вокруг, не добавлять "улучшения" без запроса.
- При работе с n8n WF — всегда проверять через MCP (`n8n_get_workflow`) актуальное состояние перед изменениями.
- Тестировать изменения виджета через curl или реальный браузер — не предполагать, что работает.
